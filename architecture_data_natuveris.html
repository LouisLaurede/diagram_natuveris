⚠️ Failles identifiées – État actuel
Doublons cross-sources
2,1 % PWA · 0,8 % CRM. Identifiants divergents entre les deux systèmes.
Impact : scores prédictifs fragmentés
Latence SAV critique
24h de délai entre ticket créé et données disponibles dans le lakehouse.
Impact : réengagement post-insatisfaction tardif
Consentement PWA partiel
ip_address, cookie_id, device_id non systématiquement documentés (Annexe 9).
Impact : risque CNIL, données exclues de l'automation
Protection transit non assurée
Chiffrement TLS 1.3 non généralisé sur les flux API PWA → lakehouse.
Impact : exposition données sensibles en transit
